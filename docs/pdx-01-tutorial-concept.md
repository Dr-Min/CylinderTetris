# PDX-01 튜토리얼 컨셉 정리

## 문서 목적

지금까지 논의한 첫 튜토리얼 방향을 고정해두기 위한 설계 문서다. 이후 UI 개선, 시작 화면 개선, 맵 화면 개선, 추가 튜토리얼 확장 시 기준으로 사용한다.

## 핵심 컨셉

- 튜토리얼 안내자는 별도의 랜덤 NPC가 아니다.
- 안내자는 코어 옆을 돌면서 같이 공격하는 기존 조력자다.
- 이름은 `PDX-01`로 간다.
- PDX-01은 방어 시스템 안에 이미 연결되어 있는 지원 파트너처럼 느껴져야 한다.
- 첫 등장에서는 크게 보여도 된다.
- 실제 전투 중에는 기존 조력자 정체성과 연결되어야 한다.
- 튜토리얼은 설명서처럼 따로 뜨는 것이 아니라, 게임 세계관 안에서 자연스럽게 유도되어야 한다.

## 성격과 말투

- 친근하고 직접적이다.
- "주인님!" 같은 표현은 가능하지만, 중요한 순간에만 가볍게 쓴다.
- 장난 캐릭터가 아니라 유능한 보조 AI 느낌을 유지한다.
- 전체 분위기는 터미널, 프로토콜, 시스템 링크 감성을 유지한다.
- 한 번에 긴 설명을 하지 않는다.
- 한 메시지에는 한 행동 또는 한 개념만 담는다.

예시:

```text
주인님, 링크 안정화 완료!
저는 PDX-01이에요. 코어 옆에서 같이 싸울게요.
/map으로 다음 구역을 확인해요.
지금은 코어를 지키는 게 우선이에요.
```

## 세계관 연결

- 플레이어는 코어를 지키고 오염된 섹터를 회수하는 오퍼레이터다.
- Safe Zone은 안정화된 기지 영역이다.
- Stage는 먼저 버텨야 하고, 이후 침투해서 회수해야 하는 적대 구역이다.
- Defense 모드에서는 생존, 코어 보호, 실드 조작, 적 압박을 배운다.
- Breach/Tetris 모드는 섹터를 정복할 수 있게 된 뒤 자연스럽게 소개한다.
- PDX-01은 이 모든 것을 "실시간 지원 안내"처럼 설명한다.

## 첫 진입 튜토리얼 흐름

1. PDX-01 링크가 수립된다.
2. Safe Zone에서 코어, 플레이어 역할, PDX-01의 역할을 짧게 소개한다.
3. PDX-01이 `/map` 입력 또는 맵 진입을 유도한다.
4. 맵 UI에서 첫 번째 진입 가능 스테이지를 화살표나 하이라이트로 보여준다.
5. 스테이지 진입 후, 코어를 지키는 것이 목표라고 알려준다.
6. 초반 방어 중 실드와 생존 압박을 짧게 안내한다.
7. 정복 가능 상태가 되면 Breach/Conquer 행동을 유도한다.
8. Breach가 시작되면 이 게임의 핵심 루프에 들어온 것을 확인시킨다.
9. 튜토리얼 완료 상태를 저장해서 기존 플레이어에게 반복 노출하지 않는다.

## UI 방향

- 터미널 느낌은 유지한다.
- 필요한 순간에만 화살표와 하이라이트를 사용한다.
- 최상단 터미널 영역을 가리지 않는다.
- 과한 모달 차단은 피한다.
- PDX-01은 통신 카드, 홀로그램 패널, 터미널 아바타 형태로 표현할 수 있다.
- 첫 등장은 크고 인상적으로 보여줄 수 있다.
- 이후 안내는 작고 상황 기반으로 보여준다.
- 모바일에서도 겹침이 없도록 처음부터 고려한다.

## 먼저 가르칠 것

- Safe Zone의 의미.
- `/map` 또는 맵 진입.
- 스테이지 선택.
- Defense 모드의 목표.
- 코어 보호.
- 실드와 생존 압박.
- Conquer/Breach 전환.

## 나중에 상황별로 가르칠 것

- Inventory.
- Upgrade.
- Mining.
- F-A Drive 알파벳 수집.
- Boss Stage.
- 아군 바이러스/조력자 고급 동작.
- 자원 경제.

## UX 규칙

- 모든 시스템을 한 번에 설명하지 않는다.
- 전투 중 대사를 난사하지 않는다.
- 모든 안내는 현재 위험을 설명하거나 다음 행동을 가리켜야 한다.
- 긴 텍스트보다 화살표와 하이라이트를 우선한다.
- 플레이어가 조작권을 빼앗겼다고 느끼면 안 된다.
- 기존 진행 데이터가 있는 플레이어에게 강제로 튜토리얼을 다시 보여주지 않는다.
- 튜토리얼 다시보기는 나중에 Safe Zone이나 터미널 명령으로 추가할 수 있다.

## 기술 배치 기준

- `js/modules/tutorial/TutorialDirector.js`가 튜토리얼 상태, 단계, 오버레이, 화살표, 완료 처리를 담당한다.
- `js/modules/GameManager.js`는 게임 상태 훅과 튜토리얼 부팅 지점을 제공한다.
- `js/modules/GameFlowMixin.js`는 Safe Zone 준비, Defense 시작, 정복 가능, Breach 시작 같은 흐름 이벤트를 제공한다.
- `js/modules/TerminalUI.js`는 튜토리얼 하이라이트 대상이 될 안정적인 UI 타겟을 제공한다.
- 스타일은 튜토리얼 전용 CSS 선택자로 분리한다.

## 남은 논의 사항

- 전체 UI 개선 이후 PDX-01의 시각적 정체성을 더 다듬는다.
- PDX-01을 초상화, 홀로그램, 스프라이트, 터미널 아바타 중 어떤 방식으로 보여줄지 결정한다.
- 튜토리얼 다시보기 기능을 넣을지 결정한다.
- F-A Drive, Mining, Boss, Inventory, Upgrade는 첫 튜토리얼 이후 상황별 안내로 확장한다.
- 시작 화면과 맵 화면은 첫 튜토리얼 기준을 바탕으로 별도 개선한다.
